Proveeduría

// SSE + reconocimiento facial + pantalla tablet

Se construyó un sistema en tiempo real para la proveeduría del gremio. El cajero usa remitero para armar el pedido. Una tablet en el mostrador muestra los productos al cliente a medida que se van cargando — sin que la página se refresque, usando SSE (una conexión persistente en lugar de requests repetidos). Los empleados se identifican por reconocimiento facial en una tercera pantalla conectada a id.puntoindigo.com, y el cajero recibe la confirmación al instante.

02
SSE del carrito — conexión persistente
SSE DevTools
Qué hace

En lugar de hacer requests HTTP cada 2 segundos, la tablet abre una sola conexión SSE que el servidor mantiene viva. Cada vez que el cajero modifica el carrito, el servidor empuja el estado actualizado por esa conexión — la tablet lo recibe en milisegundos.

Cómo probarlo

Abrir esta URL en el browser (debe quedar cargando, no terminar):

🔗 preview.remitero.puntoindigo.com/api/kiosk/cart/stream?companyId=01646687…

Luego abrir DevTools → Network y filtrar por EventSource.

Resultado esperado
  • La pestaña queda abierta indefinidamente mostrando texto plano (data: {...})
  • En Network: aparece como EventStream, no como XHR repetidos
  • Se ve un comentario : heartbeat cada ~25 segundos para mantener la conexión
DevTools → Network → filtrar: eventsource
03
Pantalla tablet cliente
Tiempo real En producción
Qué hace

Pantalla orientada al cliente, puesta en el mostrador. Muestra los productos que el cajero va cargando: nombre, cantidad, precio unitario y total. Se actualiza sola sin que nadie toque nada — el cliente puede ver su pedido evolucionar en tiempo real.

Cómo probarlo

Abrir esta URL en la tablet (o segunda pantalla del escritorio):

📱 preview.remitero.puntoindigo.com/kiosk/display?companyId=01646687…

Luego, desde remitero, crear un remito y agregar productos. Observar la tablet.

🖥️ preview.remitero.puntoindigo.com → UI cajero
Resultado esperado
  • Los productos aparecen en la tablet sin refresh de página
  • El total se actualiza al instante cada vez que el cajero agrega o quita un artículo
  • En DevTools → Network: una sola conexión EventStream activa — ningún XHR repetido
04
Terminal de identidad (kiosco facial)
SSE estado id.puntoindigo.com
Qué hace

Tablet separada en la caja donde el empleado se identifica. El cajero activa la cámara desde remitero; la terminal recibe la orden vía SSE y cambia de pantalla en menos de 1 segundo, sin que nadie tenga que tocar la tablet.

Cómo probarlo

Abrir en la tablet de identidad (requiere que el paso 01 esté completo):

📷 id.puntoindigo.com/kiosko/join/16efd110-bcc4-4446-92a3-d0de967630b4

Desde remitero, en el panel de verificación de identidad, seleccionar "Facial" y hacer clic en "Iniciar reconocimiento facial".

Resultado esperado
  • La terminal arranca mostrando pantalla idle (logo Punto Indigo)
  • Al activar desde remitero, cambia a vista facial en <1 segundo — el SSE de estado notificó el cambio
  • La cámara se activa automáticamente y comienza a buscar un rostro
05
Identificación facial end-to-end
SSE evento face-api.js
Qué hace

El empleado mira la cámara → face-api.js detecta el rostro en la tablet → el evento viaja de id a remitero vía SSE → el cajero recibe nombre y foto del empleado identificado, sin haber hecho ningún request manual.

Cómo probarlo

Con la terminal facial activa (paso 04) y el cajero esperando en remitero, el empleado mira la cámara.

Verificar en Network que no aparece /api/credito/kiosco-status — ese endpoint fue reemplazado
Resultado esperado
  • El cajero ve el nombre y foto del empleado en menos de 2 segundos desde que el rostro fue detectado
  • El panel muestra "Identidad verificada ✓" con el método "Reconocimiento facial"
  • En Network del cajero: no hay requests repetidos a /api/credito/kiosco-status
  • Una sola conexión SSE a /api/credito/kiosco-stream
06
Flujo completo proveeduría
End-to-end Crédito gremial
Qué hace

Flujo completo de una compra a crédito gremial: el empleado compra en la proveeduría, el descuento se acuerda en cuotas y se imputa al recibo de sueldo. Requiere los pasos 01–05 funcionando.

Cómo probarlo
  1. 1Cajero abre un nuevo remito en remitero
  2. 2Agrega productos — verlos aparecer en la tablet cliente en tiempo real
  3. 3En el panel de identidad, seleccionar Facial e iniciar reconocimiento
  4. 4Empleado se identifica en la terminal — cajero recibe nombre y foto
  5. 5Cajero selecciona cantidad de cuotas y guarda el remito
🖥️ preview.remitero.puntoindigo.com → UI cajero
Resultado esperado
  • Todo el flujo sin ningún loader visible para el cajero entre pasos
  • La tablet del cliente siempre muestra el estado actual del pedido
  • El remito queda guardado con empleado identificado + cuotas seleccionadas
  • La terminal vuelve a pantalla idle automáticamente al finalizar
